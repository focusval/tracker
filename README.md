# Сєвєро·3D — архів Gaussian Splats Сєвєродонецька

Статичний сайт на GitHub Pages: супутникова карта міста, на якій розставлені
тривимірні скани (Gaussian Splats), зняті дроном і зібрані в Luma AI.
Все працює з браузера телефона — без серверів, збірників і платних сервісів.
Сховище даних — сам цей репозиторій: сплат-файли лежать у `scenes/`,
позиції та налаштування — у `scenes.json`.

## Як увімкнути GitHub Pages (один раз)

1. Відкрий репозиторій на GitHub → **Settings** → **Pages**.
2. У розділі **Build and deployment** → **Source** вибери **GitHub Actions**.
3. Готово. Після кожного пушу в `main` workflow `.github/workflows/pages.yml`
   сам публікує сайт. Адреса: `https://<твій-логін>.github.io/<назва-репозиторію>/`.
4. Перший деплой можна запустити вручну: **Actions** → **Deploy GitHub Pages** →
   **Run workflow**.

Сторінка `test.html` за тією ж адресою — перевірка рендерера синтетичним кубом
(без жодних даних архіву). Якщо куб видно і грані різнокольорові — WebGL2 працює.

## Як створити fine-grained token (один раз)

Токен потрібен лише для **режиму редагування** (додавання і калібрування сцен).
Для перегляду карти токен не потрібен нікому.

1. GitHub → натисни свій аватар → **Settings** → внизу **Developer settings**.
2. **Personal access tokens** → **Fine-grained tokens** → **Generate new token**.
3. Назви як завгодно (наприклад `severo3d`), термін дії — на свій розсуд
   (коли закінчиться, просто створиш новий).
4. **Repository access** → **Only select repositories** → вибери цей репозиторій.
5. **Permissions** → **Repository permissions** → **Contents** → **Read and write**.
   Більше нічого не вмикай.
6. **Generate token** і скопіюй рядок `github_pat_…`.
7. На сайті натисни **Редагувати** і встав токен. Він зберігається тільки
   в localStorage цього браузера. Кнопка «Забути токен» видаляє його.

## Робочий цикл: від дрона до архіву

1. **Зйомка.** Обліт локації дроном: повільне коло (або два на різних висотах),
   камера дивиться на об'єкт.
2. **Luma AI.** Створи Gaussian Splat з відео. Завантаж результат у форматі SPLAT.
3. **SuperSplat** (supersplat.dev, працює і в Safari). Відкрий файл:
   - груба чистка: видали великі шматки «туману», відірвані плями неба;
   - **Export → PLY (uncompressed)**. Стиснутий (compressed) PLY сайт не прийме.
4. **Сайт.** Режим редагування → **＋ Додати скан** → вибери .ply (або .splat).
   Сцена одразу з'явиться в центрі екрана, файл конвертується у компактний
   `.splat` (32 байти на сплат — приблизно вдвічі менше за PLY) і закомітиться
   у `scenes/`.
5. **Вирівнювання.** Перетягни жовтий маркер на місце зйомки. Кнопка
   **«Авто-рівень»** сама вирівняє нахили X/Y по площині землі й підбере
   висоту (найкраще спрацьовує після ввімкнення обрізки — сфера «неба» Luma
   тоді не заважає). Масштаб і напрям на північ підганяй по будинках
   слайдерами; кнопки +90° — для сцен, що «лежать на боці».
6. **Обрізка.** Увімкни обрізку, вибери форму — **коло/еліпс** або
   **прямокутник**. Межі тягни прямо на карті за круглі ручки ↔ і ↕ на краю
   фігури (на карті показано сітку), а діапазон висот зрізає небо зверху і
   туман під землею.
7. **Гумка.** Увімкни «Гумка — стерти деталі» і малюй пальцем або мишкою по
   зайвих плямах — вони зникають (карта на цей час не рухається). Розмір
   пензля регулюється повзунком. Кнопка **«↶ Крок назад»** скасовує останній
   мазок або будь-яку зміну калібрування.
8. **Запікання.** **«Запекти зміни назавжди»** записує і обрізку, і стирання
   у менший файл; оригінал збережеться поруч як `*.orig.splat`.

Масштаб регулюється від 0.05× до **300×** — вистачає, щоб покласти скан на
супутникову карту в реальному розмірі.

**Експорт карти (ZIP).** Кнопка «Експорт карти» збирає весь сайт разом із
усіма сплатами в один `.zip` — портативна копія архіву, яку можна зберегти
про запас або відкрити на іншому хостингу (для супутникових тайлів потрібен
інтернет).
9. **Зберегти в архів.** Кнопка комітить `scenes.json`. Крапка на кнопці —
   ознака незбережених змін. Після коміту Pages передеплоїться за ~1–2 хвилини —
   доти сайт показує попередню версію.
10. **Страховка.** Кнопка «Експорт JSON» вивантажує поточний `scenes.json` файлом.

На комп'ютері (широкий екран, миша) панель редагування стає збоку праворуч,
а всі повзунки, ручки обрізки й гумка працюють мишкою.

## Ліміти, про які варто пам'ятати

- **100 МБ на файл** — жорсткий ліміт GitHub. Сайт відхиляє сплат-файли
  понад **95 МБ** із поясненням; зменшуй кількість сплатів у SuperSplat.
- Файли **до 40 МБ** комітяться через Contents API, більші — через
  Git Data API (blob → tree → commit); і там і там видно прогрес.
- **~1 ГБ на опублікований сайт** — м'який ліміт GitHub Pages. Це приблизно
  25–35 сцен по 20–40 МБ. Запікання обрізки і чистка в SuperSplat — головні
  інструменти економії. Якщо місце закінчуватиметься, найстаріші `*.orig.splat`
  можна видалити з репозиторію вручну.
- **Схована сцена не вантажиться з мережі** — перемикач видимості в
  калібруванні економить трафік і пам'ять телефона.

## Що лежить у репозиторії

```
index.html            — застосунок (перегляд + редагування)
test.html             — тест рендерера синтетичним кубом
css/style.css         — стилі (темний графіт, акцент #e5a13c)
js/app.js             — карта, завантаження сцен, режим перегляду
js/edit.js            — режим редагування (підвантажується за потреби)
js/renderer.js        — WebGL2-рендерер сплатів (кастомний шар MapLibre)
js/sorter.worker.js   — воркер сортування back-to-front
js/gsmath.js          — чиста математика (коваріації, матриці, обрізка)
js/formats.js         — парсери .ply/.splat і запис .splat
js/zip.js             — генератор ZIP (експорт карти)
js/github.js          — GitHub REST API з браузера
js/state.js, js/ui.js — стан і дрібні UI-помічники
scenes.json           — джерело правди: список сцен та їхні параметри
scenes/               — сплат-файли сцен (з'являється з першою сценою)
tests/                — Node-тести чистої логіки (node --test tests/)
```

## Тести

Локально (потрібен лише Node.js, без залежностей):

```
node --test tests/
```

Покривають: парсер PLY (на синтетичному бінарному файлі), round-trip
запису/читання .splat, коваріації, множення матриць, фільтр обрізки,
центрування хмари.
